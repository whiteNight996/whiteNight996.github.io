数据结构与算法-数组

发表于 2019-08-01

如何实现随机访问？

数组（Array）是一种线性表数据结构。它用一组连续的内存空间，来存储一组具有相同数据类型的数据。

数组的两个特性：

（1）线性表（Linear List）：数组、链表、栈、队列…与之对应的是非线性表，之所以称为非线性，是因为在非线性表中，数据表中数据并不是简单的前后关系。

（2）连续的内存空间和相同类型的数据。

正是因为这两个限制，使得数组可以支持“随机访问”，有利也有弊，这两个限制也使得数组的删除、插入操作变得低效，因为为维持连续性需要做大量的数据搬移操作。

那么问题来了，数组是如何根据下标随机获取数组元素的呢？

如何改善-低效的“插入”和“删除”

插入操作

假设数组的长度为n，现在将一个数据插入数组的第K个位置，那么插入的时间复杂度是多少呢？

如果在数组末尾插入元素，就不需要移动数据了（不考虑扩容），这时时间复杂度为O（1），如果在数组开头插入，那所有的数据都要依次向后移动一位，所以最坏时间复杂度为O（n）；因为在每个位置插入元素的概率都是一样的，所以平均时间复杂度为（1+2+3+…+n）/n = O（n）.

如果数组是有序的，我们在某个位置插入一个新的元素时候，就必须按照上述方法搬移数据。

但是，如果数组存储的数据没有任何规律，数组只是作为一个存储数据的集合。这种情况下，如果要将一个数据插入到K位置，为了避免大规模的数据搬移，我们可以：直接将第K位的数据搬移到数组元素的最后，把新的元素直接插入到第K个位置。

利用这种特殊技巧，在特定场景下，在第K个位置插入一个元素的时间复杂度就降为O（1）。

删除

和插入操作类似，如果我们要删除第K个元素，为了内存的连续性，也要搬移数据。

搬移数据会浪费大量的时间，降低效率。

实际上，在某些特殊场景下，我们不一定非得追求数组中数组的连续性。如果我们将删除操作集中在一起执行，删除的效率就会提高很多。

数组a[10]中存储了8个元素，现在要依次删除a，b，c

为了避免d、e、…这几个数据被搬移三次，我们可以先记录下已经删除的数据。每次删除操作并不是真正的搬移数据，只是记录数据已经被删除。当数组没有更多空间存储数据时，我们再执行一次真正的删除操作，这样就减少了数据搬移次数。

Tips：JVM标记清除垃圾回收算法的核心思想

很多时候我们并不是要去死记硬背某个数据结构或者算法，而是要学它背后的思想与处理技巧，这些东西才是最有价值的。

容器能否完全代替数组？

ArrayList的优势：可以将很多数组操作的细节封装起来；支持动态扩容（1.5倍）。

Tips：因为扩容操作设计内存申请和数据搬移，是比较耗时的。

如果能事先直到需要存储数据的大小，最好在创建ArrayList的时候事先指定数据大小。

经验

  • Java ArrayList无法存储基本数据类型，需要封装为包装类，而自动装箱和自动拆箱有一定的性能消耗，如果特别关注性能，或者希望使用基本数据类型，可以选用数组；
  • 表示多维数组时，使用数组会更加直观
  • 对于业务开发，容器就够用了；但如果是底层开发，数组优于容器。
